I Features

The whole stack, none of the setup.

Babel transpiles your JSX in the browser. React 18 renders it. Tailwind classes apply automatically. import { useState } from 'react' works without a bundler — named hooks bind to React.X behind the scenes, the way most playgrounds quietly hope you'll figure out yourself.

01

Auto-render

Define App or export default. We find it, render it, watch you type, re-render on change. 500 ms debounce.

02

Local library

Save files into your browser. They survive refreshes. The sidebar shows everything you've kept, sorted by recency.

03

Share by URL

One button copies a link that contains the current code, base64-encoded into the URL hash. Paste anywhere.

04

Tailwind ready

The Play CDN is loaded and scoped. Use className="flex items-center" like you would in a real project.

05

Disk in & out

Upload a .jsx from your computer; download anything you've written. Works on every browser.

06

Installable

It's a PWA. Install from the address bar — get a window with its own icon and no browser chrome.

III Questions
Where do my files live?
In your browser's localStorage. They survive refreshes and tab closes, but they're per-browser per-device. Use the share link or download to move code between machines.
Can I import npm packages?
No. There's no bundler — that's the trade for "no setup". import from react and react-dom is rewired to use the loaded globals; everything else is silently stripped. For real projects with packages, use a Vite project on your machine.
Is it really free?
Yes. There's no backend to pay for — the whole site is static files. It runs in your browser. The author serves it from GitHub Pages at zero cost.
Will my code be sent anywhere?
No. Compilation and rendering happen entirely in your browser. Network requests go only to public CDNs (cdnjs, Google Fonts) for the libraries themselves.
Can I host my own copy?
Yes — clone the repo, push to your own GitHub Pages or any static host. The whole site is one folder of HTML/CSS/JS.
